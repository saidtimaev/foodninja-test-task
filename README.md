# Тестовое задание: Сервис сокращения ссылок (Food Ninja)

## Архитектурные особенности и оптимизация

Проект спроектирован с учетом высоких нагрузок и включает в себя следующие архитектурные решения:

* **Разделение контекстов:** Публичная часть (обработка редиректов) полностью изолирована от тяжелой панели управления Filament. Редиректы обрабатываются легковесным `RedirectController`.
* **Кеширование (Redis):** Для исключения повторных запросов к реляционной БД при высокой частоте переходов, соответствие пар `короткий_код => оригинальный_url` кэшируется в Redis.
* **Асинхронный сбор статистики (Очереди):** Логирование переходов (IP-адрес, таймстамп) вынесено в фоновую задачу `CreateLinkVisitJob` через Laravel Queues. Это гарантирует, что пользователь перенаправляется мгновенно, не дожидаясь записи логов в базу данных.
* **Индексация и оптимизация БД:** На уровне миграций спроектированы индексы для полей выборки (`code`, `user_id`), сортировки (`created_at`) и мягкого удаления (`deleted_at`), обеспечивая стабильную скорость работы PostgreSQL.
* **Полная изоляция данных:** Реализована строгая изоляция данных на уровне моделей (Eloquent Global Scopes). Каждый авторизованный пользователь видит, редактирует и анализирует историю переходов **исключительно своих** ссылок.

## Стек технологий

- **Фреймворк:** Laravel 13
- **Админ-панель:** Filament v3
- **Кеш / Очереди:** Redis
- **База данных:** PostgreSQL 16
- **Окружение:** Docker (Кастомный Dockerfile + Docker Compose)

## Быстрый запуск (Docker Compose)

Проект полностью контейнеризирован и настроен для автоматического развертывания "из коробки". Все рутинные процессы — от генерации ключей и установки зависимостей до накатывания миграций и запуска фоновых воркеров — Docker берет на себя.


### 1. Клонирование репозитория 
```bash
git clone [https://github.com/saidtimaev/foodninja-test-task.git](https://github.com/saidtimaev/foodninja-test-task.git)
cd food-ninja-test
cp .env.example .env
```

### 2. Запуск проекта в одну команду

```bash
docker compose up -d --build
```

### Доступ к приложению

Перейдите по адресу http://localhost:8000 что бы попасть на главную страницу
